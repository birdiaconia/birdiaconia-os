# Birdiaconia Channel Operating Model v0.1

## 문서 목적

이 문서는 버디아코니아가 여러 디지털 채널을 하나의 운영체계로 연결해 사용하는 방식을 정의한다.

버디아코니아는 하나의 웹사이트나 하나의 플랫폼에 모든 기능을 몰아넣지 않는다. 각 채널이 가장 잘하는 역할을 맡고, Vercel Workspace는 그 채널들을 가볍게 연결하고 통제하는 내부 운영실로 사용한다.

이 문서의 핵심은 다음 문장으로 요약된다.

> “인스타그램은 보여주고, 네이버 플레이스는 검색되며, 네이버 카페는 사람을 모으고, 폼과 AppSheet는 입력을 받고, 시트는 저장하고, Vercel은 통제하며, AI는 정리하고 출력한다.”

---

## 1. 기본 원칙

버디아코니아의 멀티채널 운영 모델은 다음 원칙을 따른다.

1. **채널은 복제하지 않고 연결한다.**
   - Vercel은 네이버 카페, 인스타그램, 네이버 플레이스의 기능을 다시 만들지 않는다.
   - 이미 잘 작동하는 외부 채널은 해당 채널의 장점을 그대로 사용한다.

2. **Vercel은 운영 통제실이다.**
   - Vercel Workspace는 내부 운영자가 전체 흐름을 확인하고 다음 행동을 판단하는 가벼운 cockpit이다.
   - 게시판, 커뮤니티, 검색 노출, 개인정보 저장소가 아니다.

3. **개인정보는 Vercel에 저장하지 않는다.**
   - 신청자, 회원, 자원봉사자, 후원자 등 개인 데이터는 Vercel에 저장하지 않는다.
   - 개인정보가 필요한 입력은 Google Forms 또는 AppSheet로 받고, 저장은 비공개 Google Sheets에서 관리한다.

4. **GitHub는 시스템의 설계와 변경 이력이다.**
   - 운영 원칙, 문서, 코드, 변경 기록은 GitHub에 남긴다.
   - GitHub는 버디아코니아 OS가 어떻게 진화했는지 확인하는 공식 기록장이다.

5. **AI는 정리와 출력의 엔진이다.**
   - AI는 흩어진 입력을 정리하고, 보고서·제안서·발표자료·문서 산출물을 만든다.
   - AI는 최종 의사결정자가 아니라 운영자를 돕는 분석 및 작성 보조자다.

## 2. 채널별 역할

| 채널 | 핵심 역할 | 사용 목적 | 하지 않는 일 |
|---|---|---|---|
| Vercel Workspace | 가벼운 내부 운영 cockpit | 운영 현황 확인, 링크 연결, 다음 행동 통제 | 커뮤니티 게시판, 개인정보 저장, 공개 홍보 채널 복제 |
| Naver Cafe | 커뮤니티와 사람 모으기 | 공지, 자원봉사 모집, 활동글, 회원 소통 | 내부 데이터베이스 역할 |
| Instagram | 시각 홍보와 짧은 업데이트 | 사진, 짧은 소식, 브랜드 스토리텔링 | 신청 접수나 장기 문서 보관 |
| Naver Place | 공개 검색 존재감 | 위치, 연락처, 신뢰 형성, 검색 유입 | 커뮤니티 운영이나 내부 업무 관리 |
| Google Forms / AppSheet | 신청과 운영 입력 | 신청서, 활동 신청, 현장 입력, 운영 양식 | 장기 아카이브 단독 역할 |
| Google Sheets | 비공개 데이터 저장 | 신청 데이터, 운영 데이터, 내부 관리 목록 | 공개 게시판 또는 홍보 채널 |
| Google Drive | 문서와 미디어 아카이브 | 사진, 영상, 계약서, 보고서, 원본 파일 보관 | 실시간 운영 통제 화면 |
| GitHub | 시스템 설계와 변경 이력 | OS 문서, 코드, Issue, Pull Request, 변경 기록 | 회원 커뮤니케이션 채널 |
| AI | 정리·분석·보고·출력 | 보고서, 제안서, PDF, PPT, Word 초안 생성 | 사람의 승인 없는 최종 결정 |

## 3. 채널 운영 구조

버디아코니아의 채널은 다음 흐름으로 함께 작동한다.

```text
외부 노출
Instagram / Naver Place
↓
사람 모으기와 소통
Naver Cafe
↓
신청과 운영 입력
Google Forms / AppSheet
↓
비공개 데이터 저장
Google Sheets
↓
문서와 미디어 보관
Google Drive
↓
운영 통제와 연결
Vercel Workspace
↓
설계, 코드, 변경 기록
GitHub
↓
정리, 분석, 보고서 출력
AI
```

이 흐름은 단방향으로만 움직이지 않는다. 예를 들어 AI가 활동 기록을 정리해 보고서를 만들면, 운영자는 그 결과를 바탕으로 네이버 카페 공지, 인스타그램 게시물, 제안서, 내부 문서로 다시 나누어 활용할 수 있다.

---

## 4. Vercel Workspace의 범위

Vercel Workspace는 버디아코니아 OS의 내부 운영 화면이다. 목적은 모든 기능을 직접 제공하는 것이 아니라, 운영자가 여러 채널을 놓치지 않도록 연결하고 통제하는 것이다.

### Vercel이 해야 하는 일

- 오늘 확인해야 할 운영 항목을 보여준다.
- 네이버 카페, 인스타그램, 네이버 플레이스, Google Forms, AppSheet, Google Sheets, Google Drive, GitHub로 이동하는 링크를 정리한다.
- 각 채널의 역할과 현재 상태를 짧게 보여준다.
- 운영자가 다음 행동을 판단할 수 있도록 요약 정보를 제공한다.
- GitHub 문서와 연결해 운영 원칙을 확인할 수 있게 한다.

### Vercel이 하지 말아야 하는 일

- 네이버 카페를 대체하는 커뮤니티 게시판을 만들지 않는다.
- 인스타그램을 대체하는 피드나 시각 홍보 채널을 만들지 않는다.
- 네이버 플레이스를 대체하는 위치·검색·리뷰 시스템을 만들지 않는다.
- 개인정보를 저장하지 않는다.
- 회원 데이터베이스를 만들지 않는다.
- 인증 시스템을 만들지 않는다.
- 운영 입력 양식을 자체 데이터베이스와 연결해 저장하지 않는다.

### 핵심 규칙

> Vercel은 운영체계를 **통제하고 연결**한다. Vercel은 사람을 모으는 공간도, 공개 홍보 채널도, 개인정보 저장소도 아니다.

## 5. Naver Cafe의 역할

네이버 카페는 버디아코니아가 사람을 모으고 관계를 유지하는 중심 커뮤니티 채널이다.

- 알려진 네이버 카페 주소: <https://cafe.naver.com/birdiaconia>

네이버 카페는 다음 역할을 맡는다.

- 공식 공지 게시
- 자원봉사자 모집
- 활동 후기와 현장 소식 게시
- 회원과 관심자 소통
- 반복적으로 안내해야 하는 커뮤니티 정보 축적

Vercel은 네이버 카페의 게시판 기능을 복제하지 않고, 운영자가 필요한 시점에 카페로 이동할 수 있도록 연결한다.

---

## 6. Instagram의 역할

인스타그램은 버디아코니아의 시각적 인상과 짧은 소식을 전달하는 채널이다.

주요 역할은 다음과 같다.

- 활동 사진과 짧은 영상 공유
- 브랜드 분위기와 스토리텔링 전달
- 짧은 업데이트와 캠페인 홍보
- 외부 관심자에게 버디아코니아의 이미지를 보여주기

인스타그램은 신청 접수, 장기 문서 보관, 운영 데이터 저장을 담당하지 않는다.

---

## 7. Naver Place의 역할

네이버 플레이스는 버디아코니아가 외부에서 검색되고 신뢰를 얻기 위한 공개 정보 채널이다.

주요 역할은 다음과 같다.

- 위치 정보 제공
- 연락처와 기본 운영 정보 제공
- 네이버 검색에서 발견될 수 있는 존재감 확보
- 외부 방문자와 보호자, 협력자가 기본 신뢰를 형성하도록 돕기

네이버 플레이스는 커뮤니티 운영이나 내부 업무 관리 도구가 아니다.

---

## 8. Google Forms, AppSheet, Sheets, Drive의 역할

Google 도구들은 입력, 저장, 보관을 분리해 사용한다.

### Google Forms / AppSheet

Google Forms와 AppSheet는 신청과 운영 입력을 받는 창구다.

- 자원봉사 신청
- 프로그램 신청
- 현장 기록 입력
- 운영 체크리스트 입력
- 반복 업무 입력 화면

### Google Sheets

Google Sheets는 비공개 운영 데이터 저장소다.

- 신청자 목록
- 활동 참여 기록
- 운영 관리 데이터
- 내부 확인용 표

개인정보나 민감한 운영 데이터는 공개 화면이나 Vercel에 저장하지 않고, 접근 권한이 관리되는 Google Sheets에서 다룬다.

### Google Drive

Google Drive는 문서와 미디어의 아카이브다.

- 활동 사진과 영상 원본
- 보고서와 제안서 파일
- 계약서와 행정 문서
- 발표자료, PDF, Word, PPT 산출물

Drive는 자료 보관소이며, 실시간 운영 cockpit은 Vercel이 맡는다.

---

## 9. GitHub의 역할

GitHub는 버디아코니아 OS의 설계, 코드, 변경 이력을 관리하는 공간이다.

GitHub에는 다음을 남긴다.

- 운영 원칙 문서
- 시스템 설계 문서
- Vercel Workspace 코드
- Issue 단위의 작업 기록
- Pull Request 단위의 검토 기록
- 변경 이력과 결정의 흔적

GitHub는 커뮤니티 채널이 아니다. GitHub는 버디아코니아의 운영체계가 어떤 판단과 변경을 거쳐 만들어졌는지 확인하는 내부 시스템 기록장이다.

---

## 10. AI의 역할

AI는 버디아코니아 운영에서 정리, 분석, 작성, 출력의 역할을 맡는다.

AI가 할 수 있는 일은 다음과 같다.

- 회의 내용과 대화 정리
- 활동 기록 요약
- 운영 데이터 분석 보조
- 보고서 초안 작성
- 사업 제안서 초안 작성
- 공지문과 홍보 문안 초안 작성
- PDF, PPT, Word 산출물 구성
- GitHub Issue 또는 문서 초안 정리

AI가 하지 말아야 하는 일은 다음과 같다.

- 사람의 승인 없이 최종 의사결정을 확정하지 않는다.
- 개인정보를 불필요하게 복사하거나 외부 공개 문서에 포함하지 않는다.
- 운영 원칙을 임의로 바꾸지 않는다.
- 외부에 공개될 내용을 사람의 검토 없이 확정하지 않는다.

## 11. 운영 예시

### 자원봉사 모집

1. 인스타그램에 짧은 홍보 게시물을 올린다.
2. 네이버 카페에 자세한 모집 공지를 올린다.
3. 신청은 Google Forms 또는 AppSheet로 받는다.
4. 신청 데이터는 Google Sheets에 저장한다.
5. 관련 사진과 문서는 Google Drive에 보관한다.
6. Vercel Workspace에는 모집 상태와 관련 링크만 연결한다.
7. GitHub에는 운영 기준이나 양식 변경이 필요한 경우 문서 변경으로 기록한다.
8. AI는 신청 현황 요약, 공지 초안, 결과 보고서 초안을 만든다.

### 활동 기록 정리

1. 활동 사진과 영상은 Google Drive에 보관한다.
2. 활동 글은 네이버 카페에 게시한다.
3. 핵심 장면은 인스타그램에 짧게 공유한다.
4. 운영 데이터는 Google Sheets에 정리한다.
5. AI가 활동 보고서와 제안서용 요약을 만든다.
6. Vercel Workspace는 관련 자료 링크와 후속 작업만 보여준다.

---

## 12. 최종 기준

버디아코니아의 멀티채널 운영 모델은 다음 기준을 지킨다.

- 공개 홍보는 Instagram과 Naver Place가 맡는다.
- 사람 모으기와 커뮤니케이션은 Naver Cafe가 맡는다.
- 입력은 Google Forms와 AppSheet가 맡는다.
- 저장은 Google Sheets가 맡는다.
- 문서와 미디어 보관은 Google Drive가 맡는다.
- 운영 통제와 연결은 Vercel Workspace가 맡는다.
- 설계와 변경 이력은 GitHub가 맡는다.
- 정리, 분석, 보고서와 제안서 출력은 AI가 맡는다.

따라서 Vercel은 다른 채널을 대체하지 않는다. Vercel은 버디아코니아 OS의 가벼운 내부 cockpit으로서 여러 채널을 연결하고, 운영자가 전체 흐름을 통제하도록 돕는다.
